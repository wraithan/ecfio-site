ecfio \ EK-fee-oh \

A framework of utilities, documentation, and modules that make hobbyist hardware development easier, faster, and generally more fun.

This is a work in progress, the first iteration of the ecfio commandline tool is not yet available.

Description

Background

Software development for hobbyist hardware has not evolved nearly as much as the hardware itself has in the last few years. Arduino IDE makes it easy for people on any platform to get started with very basic applications, but once you start doing more complex things you are limited to what is already there, or if you feel bold, going through the complex process needed to install headers, code, and libraries on top of the Arduino IDE's files.

Improving the situation

The first aim will be clean up the build systems. The plan is to leverage dotc and ino as a starting point. ecfio will wrap these two tools (and maybe others) to streamline project creation and development.

Next, will be moving a few of the core modules from the arduino IDE into dotc modules so they are tracked in their own repos and able to be updated outside of updating the IDE. This will be a test of the dotc system and whether this is truly feasible. Along with the moves into their own modules, this will be the start of documenting exisiting modules.

Goal

The eventual goal will be to have an ecosystem of modules, built on open source tools, that have documentation and are easily available.

Usage

ecfio init

Initializes your project, creating the base directory structure and files.

ecfio build

Takes your project, preprocesses it, and then builds the .hex file.

ecfio push

Takes the .hex file and pushes it to your device
