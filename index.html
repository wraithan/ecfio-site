ecfio \ EK-fee-oh \

A framework of utilities, documentation, and modules that make hobbyist hardware development easier, faster, and generally more fun.

Description

Currently, the environment for hobbyist electronics is hostile. There is plenty of open source hardware, but the tooling is pretty exclusive to random GUI tools, copy and paste code, half translated data sheets, and scattered blog posts. The community deserves better. Modern development practices and good documentation is really what is needed, that is what ecfio will provide.

The first aim will be clean up the build systems. The plan is to leverage dotc and ino as a starting point. Once there are 2-3 projects using this system, the most common commands will be documented. The result of this should be a couple one liners for building projects, pushing them to boards, and perhaps things like running test suites.

Next, will be moving a few of the core modules from the arduino IDE into dotc modules so they are tracked in their own repos and able to be updated outside of updating the IDE. This will be a test of the dotc system and whether this is truly feasible. Along with the moves into their own modules, this will be the start of documenting exisiting modules.

Notes

Not all of the utilities will be written by, or even owned by, the ecfio project. In fact it is almost better that they aren't. The goal of this will be to document the interoperability, build a base ecosystem of modules, build any glue tools needed, and promote it all.
