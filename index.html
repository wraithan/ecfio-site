ecfio \ EK-fee-oh \

An ecosystem of utilities, documentation, and modules that make software development for hobbyist hardware easier, faster, and generally more fun.

Description

Software development for hobbyist hardware has not progressed nearly as much as the hardware itself has in the last few years. Arduino IDE makes it easy for people on any platform to get started and that is valuable, but advanced use cases are harder. Currently, to use libraries you find on the internet, you are left copy and pasting them into your project, or putting the files into multiple directories in the IDE's directory.

The first aim will be clean up the build system. The plan is to leverage dotc and ino as a starting point. ecfio will wrap these two utilities (and maybe others) to streamline project creation and development.

Next, will be move a few of the core modules from the Arduino IDE into dotc modules. They will be tracked in their own repos and able to be updated outside of the IDE. The module repos will include documentation for that module, making it a complete unit. This will be a test of the dotc system and whether this project is truly feasible.

The eventual goal will be to have an ecosystem of modules, built on open source tools, that have documentation and are easily installable.

Example Commands

This is a work in progress.

ecfio init

Initializes your project, creating the base directory structure and files.

ecfio build

Takes your project, preprocesses it, and then builds the .hex file.

ecfio push

Takes the .hex file and pushes it to your device.
